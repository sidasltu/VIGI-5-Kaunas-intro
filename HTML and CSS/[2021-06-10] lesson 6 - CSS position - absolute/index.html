Primary HTML elements

There are over 100 HTML elements, but most of them are out-dated or rarely used. There are about 20 primary/common elements which every web developer should know and use correctly.

HTML intro

HTML element is a special syntax, which purpose is to structure page and it's content. Most HTML elements have opening and closing tag. Opening tag is wrap in diamond quotes:
<HTML tag name>
Closing tag is writen similary, but with slash before tag name:
</HTML tag name>

Everything between opening and closing tag is interpreted as children components.

Attributes are additional setting for HTML element. They should be writen in element's opening tag. There are common attributes for all elements like: class, style, etc.. Some HTML elements have specific attributes, for instance:
<img> has 'alt' and 'src' attributes
<a> has 'href' and 'target' attributes
... etc.

Inline Elements

Inline elements are interpreted as text. These elements can't have spacing settings. This kind of elements are positioned from left to right, and only if there are no more space, they are placed in the next row.

Most common style properties for inline elements

  • color - text color
  • text-align - horizontal alignment
  • vertical-align - vertical alignment
  • letter-spacing - spacing between letters
  • line-height - height of one line
  • ... etc.

All typography(text) style properties are applied to element and all of it's descending children!

Most common inline elements

  • span - basic inline element for text
  • a - reference element to another page or page section
  • img - image
  • strong - element to emphasize(bold) text
  • em - element to make inner text italic(oblique)
  • ... etc.

Inline-block elements

Inline block elements behave as inline elements, but can have spacing and dimension properties. Inline style properties are also applied for inline-block elements.
The main diference is that content in inline-block elements is grouped and can't be moved to next line seperately.

Most common inline-block elements

  • input - user input field
  • text-area - user input text area
  • select - user input with options

Block elements

Block elements reserves all perent element width. Even if content width in block element is smaller then full parent element width, other elements can't be in the same row.

Most common block elements

  • section - is used for page sections, most global elements
  • header - elements used for page, or other components top view
  • footer - elements used for page, or other components bottom view
  • article - elements used for multiple reusable components and gadgets
  • main - main elements, in which should consist of most important page section
  • div - standard block element
  • etc...

Position

Position property adds additional positioning functionality to elements. That functionality is implemented by using CSS properties: top, right, bottom and left
Static is default position type, and direction properties havo no effect.

Position relative

Position relative can shift element using direction properties

static position: relative; top: 20px; left: 60px; static static

Position absolute

Position absolute element serches for NON STATIC element in his parent line. If it finds one - he shifts from it's contours by properties: left, right, bottom, top.
If element with NON STATIC position is not found, it shifts from viewport.
